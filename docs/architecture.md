# UAV-USV 平台总体架构说明

## 项目定位

本项目面向无人机、无人艇协同感知与围捕任务，目标是形成一个可演示、可联调、可扩展的半实物仿真验证平台。

平台不是普通后台管理系统，而是围绕“任务配置、状态监控、通信链路、协同围捕、实验评估”的任务控制系统。

## 对齐 PPT 的五层架构

### 第一层：仿真与实物执行层

职责：

- Unity3D 虚拟场景
- Gazebo / ROS2 仿真
- UAV / USV / 岸基站 / 半实物接口
- 后续真实无人艇、无人机状态接入

当前状态：

- WSL2 中已有 ROS2/Gazebo 仿真。
- Unity 已能通过 WebSocket 同步 UAV/USV 位姿。
- 平台已预留 ROS/Unity 运行状态读取和启动控制框架。

后续重点：

- 继续完善 Unity 海面、无人艇、无人机、目标和围捕过程表现。
- 逐步对接真实设备或半实物接口。

### 第二层：通信与认知层

职责：

- ROS/ROS2 Topic、Service、Action
- WebSocket / REST / MQTT / UDP/TCP
- 多平台状态共享
- 网络状态监测
- 后续智简通信、语义摘要和链路调度

当前状态：

- 后端已作为 WebSocket 客户端接入 ROS 位姿流。
- Unity 已通过心跳接口上报状态。
- 运行监控页面已显示节点在线、心跳、坐标和链路状态。

后续重点：

- 建立通信网络监控模块。
- 记录链路延迟、丢包率、带宽、断连事件和关键消息。

### 第三层：智能算法层

职责：

- 目标检测
- 融合定位
- 意图识别
- 任务分配
- 路径规划
- 围捕控制

当前状态：

- 还未正式接入 Python 算法服务。
- 前后端仅预留任务、状态和评估数据模型方向。

后续重点：

- 以 REST 或 Socket 接入 Python 算法服务。
- 前端先做算法结果展示和任务配置，后端封装算法接口。

### 第四层：业务服务层

职责：

- 设备管理
- 任务管理
- 接口管理
- 日志告警
- 实验评估
- 用户管理
- 数据管理

当前状态：

- Spring Boot 后端已完成登录、设备管理、运行监控、运行控制基础模块。
- 已使用 MySQL、Flyway、JPA 和 MVC 分层结构。

后续重点：

- 增加任务管理、任务事件、轨迹点、通信链路、实验评估等业务表。
- 建立可追踪的任务生命周期。

### 第五层：展示交互层

职责：

- Vue 前端任务控制台
- Unity3D 三维态势平台
- 二维态势、任务配置、状态监控、通信监控、围捕展示、评估回放

当前状态：

- Vue 已完成登录、系统总览、设备管理、运行监控。
- 系统总览、设备管理、运行监控已按任务态势平台风格优化。

后续重点：

- 按 PPT 七类界面继续补齐：
  - 综合态势显示
  - 任务配置
  - 智能体状态监控
  - 感知结果
  - 通信网络
  - 围捕过程
  - 实验评估与回放

## 当前平台边界

当前前后端主要负责：

- 资产登记
- 运行状态读取
- 任务控制台界面
- ROS/Unity 状态桥接
- 后续任务管理和评估数据承载

暂不直接承担：

- 真实飞控安全控制
- 复杂围捕算法计算
- Unity 场景渲染核心逻辑
- ROS 仿真核心模型修改

这些内容应通过接口接入，保持模块边界清晰。
