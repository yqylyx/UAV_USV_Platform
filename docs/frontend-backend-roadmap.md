# 前后端模块规划

## 已完成模块

- 用户表 `app_user`
- Spring Security 会话认证
- CSRF 保护
- 初始管理员 `admin`

- 登录页面

### 系统总览

定位：

- 面向演示和任务态势的入口页面。
- 不再显示后端服务、MySQL 等开发状态。

当前内容：

- 三无人机、三无人艇围捕目标的态势总览。
- 协同编队、任务闭环、链路健康、五层架构映射。

### 设备管理

后端：

- 设备 CRUD
- 分页查询
- 类型筛选
- 状态筛选
- 软删除

前端：

- 海空协同任务资产注册台
- 设备卡片矩阵
- 新增、编辑、删除确认
- 查询、筛选、分页

当前显示设备：

- UAV
- USV

### 运行监控

后端：

- ROS WebSocket 位姿接入
- Unity 心跳接入
- 节点状态缓存和心跳超时判断
- SSE 状态推送
- 运行控制状态接口

前端：

- 运行链路指挥条
- 在线率、异常关注、通信/可视化状态
- UAV、USV、ROS、Unity 在线卡片
- 实时节点卡片
- 心跳、坐标、数据源、IP/端口显示

## 下一阶段优先模块

### 1. 任务管理 / 任务配置

目标：

- 新建协同任务。
- 选择 UAV、USV、目标。
- 配置任务类型、环境参数、通信参数、传感器能力。
- 形成后续运行监控、围捕展示、评估回放的业务对象。

建议优先级：最高。

### 2. 任务阶段与事件日志

目标：

- 记录任务生命周期事件。
- 支持目标发现、协同分配、路径跟踪、围捕收缩、完成、异常。
- 后续接 ROS/Unity/算法服务上报事件。

建议优先级：高。

### 3. 通信网络监控

目标：

- 展示 ROS、Unity、算法服务、设备节点的链路拓扑。
- 记录心跳、延迟、丢包率、带宽、关键消息。

建议优先级：高。

### 4. 协同围捕过程展示

目标：

- 展示目标、UAV、USV、包围圈、围捕阶段、成功率。
- 初期可使用二维模拟数据。
- 后续接 ROS 位姿和算法服务结果。

建议优先级：中高。

### 5. 实验评估与回放

目标：

- 记录任务过程轨迹与事件。
- 输出完成时间、捕获率、通信开销、路径长度、能耗、协同效率。
- 支持回放时间轴和报告导出。

建议优先级：中。

## 后端模块规划

建议新增模块：

```text
module
├─ mission          # 任务管理
├─ missionevent     # 任务事件与阶段
├─ communication    # 通信链路监控
├─ encirclement     # 围捕过程数据
└─ evaluation       # 实验评估与回放
```

每个模块保持：

```text
controller
dto/request
dto/response
entity
repository
service
service/impl
```

## 接口原则

- 前端只调用 Spring Boot 后端，不直接操作数据库。
- 后端对 ROS/Unity/算法服务做适配和数据归一。
- ROS/Unity/算法服务可以上报状态和事件，但不应直接驱动前端状态。
- 先做只读监控和数据记录，再做启动、停止、任务下发等控制能力。
